Overview

Test any IOS native, hybrid, or mobile web application using the selenium API. IOS automation becomes just another driver. You can reuse the helper classes from your web tests to create data, and follow the same designs you're used to ( page object etc ).

High level features :

  • Implements the JSON wire protocol
  • can use grid to manage device / simulator farm
  • first class support for localized native apps ( 1 test works for all the localized versions )
  • no need to change the source
  • no need to jailbreak the phone
  • no ios-driver app to install on the phone

Scaling

ios-driver has been built with scaling in mind. GUI based tests aren't fast enough to provide feedback if they are run sequentially. By following the webdriver protocol, ios-driver can be used as a selenium grid node, and the tests can be run in parallel on the same architecture as the one already running the web tests.
