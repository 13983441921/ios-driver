ios-driver

IOS automation for native, hybrid and mobile web.

Running ios-driver on the Grid

You can add ios-driver as a node to a running grid hub. In order to connect to a grid, you can specify the below command line parameters when running ios-driver:

java -jar ios-driver.jar -hub http://127.0.0.1:4444/grid/register -host 127.0.0.1 -port 5555

You can also specify a custom proxy to use on the grid, for example:

java -jar ios-driver.jar -hub http://127.0.0.1:4444/grid/register -proxy org.uiautomation.ios.grid.IOSRemoteProxy -host 127.0.0.1 -port 5555

IOSRemoteProxy

IOSRemoteProxy is a custom remote proxy which is designed to work with ios-driver nodes. In particular it polls the node to see if the capabilities have been updated, this will allow you to use ios-driver as part of your CI process where new version of apps could be added to ios-drivers capabilities. Have a look at the -folder parameter option if you'd like to dynamically pick up new apps. If IOSRemoteProxy discovers new capabilites on the node, it will mark the node as down, wait for any running tests to complete and will then re-register the node with the updated capabilities making them available for testing.

Building IOSRemoteProxy

> cd ios-driver/grid/

> mvn package

copy the resulting ios-grid.jar file from ios-driver/grid/target to the location where you run the hub. 
Run hub like:

> java -cp ios-grid.jar:selenium-server-standalone.jar org.openqa.grid.selenium.GridLauncher -role hub
