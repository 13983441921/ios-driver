Errors launching ios-driver

Can't load metadata

If you see this error 
  Exception in thread "main" org.openqa.selenium.WebDriverException: cannot load the metadata from the Info.plist file for /Applications/Xcode.app/Contents/Developer/Platforms/iPhoneSimulator.platform/Developer/SDKs/iPhoneSimulator6.1.sdk/Applications/MobileSafari.app
you need to give ios-driver access to write the MobileSafari info.plist 
  sudo chmod 666 /Applications/Xcode.app/Contents/Developer/Platforms/iPhoneSimulator.platform/Developer/SDKs/iPhoneSimulator*.sdk/Applications/MobileSafari.app/Info.plist

Exiled SDKs warning

The exiledSDKs folder is used to temporarilly store the most recent SDKs allowing us to run against an older version. If you get warnings about this folder, firstly make sure the following location exists 
  sudo mkdir /Applications/Xcode.app/Contents/Developer/Platforms/iPhoneSimulator.platform/Developer/exiledSDKs
and ios-drvier has read write permissions to the location 
  sudo chmod -R +rw /Applications/Xcode.app/Contents/Developer/Platforms/iPhoneSimulator.platform/Developer/exiledSDKs

Developer Tools Access needs to take control


To stop this message being displayed you need to udpate /etc/authorization. 
  Find the rule system.privilege.taskport. 
  Under the line <key>allow-root</key>, change <false/> to <true/>. 

  It should look something like:
  <key>system.privilege.taskport</key>
    <dict>
        <key>allow-root</key>
        <true/>
